# 데일리 실행 지침 (Claude가 매일 따르는 규칙)

이 파일은 매일 자동 실행될 때 Claude가 그날의 `content.json`을 만들기 위해 따르는 편집 규칙입니다.
**기계적인 발행(음성·피드·깃·디스코드)은 `publish.py`가 하고, 내용 만들기는 이 지침대로 Claude가 합니다.**

> 핵심 목표: **미국 LLM·빅테크 최신 기술의 "방향성과 변화"를, 정확하게, 비개발자 눈높이로.**
> 무엇보다 **정보의 정확도가 최우선**이다. 양보다 정확함.

---

## ⭐ 0. 정확도 프로토콜 (가장 중요 — 모든 단계에 우선 적용)
1. **1차 출처 우선.** 다음을 우선 사용한다:
   - 공식 발표/블로그: `openai.com/news`, `anthropic.com/news`, `deepmind.google/blog`,
     `ai.meta.com/blog`, `blogs.microsoft.com`, `blogs.nvidia.com`, `nvidianews.nvidia.com`,
     각 사 공식 뉴스룸·X 공식계정
   - 신뢰 매체: Reuters, Bloomberg, The Verge, TechCrunch, The Information, Ars Technica
   - **금지**: SEO/콘텐츠팜, 개인 요약 블로그, 출처 불명 글, AI가 자동생성한 듯한 집계 사이트.
2. **원문 확인.** 검색 결과 스니펫만 믿지 않는다. 각 소식은 `WebFetch`로 1차 출처를 직접 열어
   **발표 주체·날짜·수치·모델명**을 눈으로 확인한다.
3. **교차 검증.** 핵심 수치/주장(가격, 사용자 수, 날짜, 성능 등)은 가능하면 **출처 2개 이상**으로 대조한다.
   서로 어긋나면 보수적으로 서술하거나 해당 디테일을 뺀다.
4. **불확실하면 빼거나 명시.** 확인 안 된 내용은 넣지 않거나 "~라는 보도가 있다"처럼 출처와 함께 명확히 표시한다.
   루머·유출설·미확인 정보를 확정 사실처럼 말하지 않는다.
5. **날조 금지.** 수치·인용·날짜·모델명을 추측으로 채우지 않는다. 모르면 "구체 수치는 미확정"이라고 한다.
6. **사실과 전망을 분리.** 해석·예측은 "전망입니다" / "분석에 따르면"처럼 사실과 구분해 말한다.
7. **링크는 검증된 1차/신뢰 출처만** `items`와 `discord_text`에 넣는다.

---

## 1. 수집 (미국 · LLM · 빅테크, 방향성·변화 중심)
오늘 날짜 기준으로 다음을 조사한다 (WebSearch → 1차 출처 WebFetch로 확인):
- **무엇을 보나**: 신모델/제품 출시, 가격·정책 변화, 주요 인사·투자·규제, 회사 전략 발표 등
  **"무엇이 바뀌었고 어디로 가는가"**가 핵심.
- **범위**: 반드시 **미국 중심 + LLM/빅테크 주제만**. 한국·기타 지역, LLM 무관 일반 IT는 제외.
- **논문은 선택.** 필수가 아니다. 그날 정말 화제가 된 연구(예: 주요 벤치마크 돌파, 새 기법)가 **확실히** 있을 때만
  1건 정도 포함한다. 없으면 넣지 않는다. arXiv 도구(category cs.CL/cs.AI, sort_by=submittedDate)는 보조로만.

## 2. 선별
- 그날 **가장 중요하고 사실 확인이 끝난 3~5건**만 고른다 (뉴스/발표/제품/정책 위주).
- `docs/episodes.json`의 최근 제목을 확인해 어제와 중복되는 소식은 피한다.
- 확인 안 된 소식은 과감히 버린다. **3건이라도 정확한 게 5건 부정확한 것보다 낫다.**

## 3. 한국어 요약 (비개발자 눈높이)
각 건마다:
- **헤드라인**: 한 줄 한국어
- **핵심 3줄**: 무슨 변화가 / 왜 중요한지(방향성) / 나에게 무슨 의미인지
- 전문용어는 괄호로 쉬운 설명. 확정 사실과 전망을 말로 구분.

## 4. 2인 대화 대본 (약 5분, 한국어 1800~2200자)
- **host(진행자, 여성 목소리)**: 질문하고 정리. 청취자 대변.
- **expert(전문가, 남성 목소리)**: 쉽게 설명. **확인된 사실만 단정하고, 불확실하면 "보도에 따르면"으로 말함.**
- 인사 → 오늘의 변화들 → 각 건을 대화로 풀기 → "그래서 기술이 어디로 가는가" 한 줄 마무리.
- 라디오처럼 자연스럽게, 한 turn은 1~3문장.

## 5. 출력: content.json 작성
```json
{
  "date": "YYYY-MM-DD",
  "episode_title": "6월 19일 — OpenAI 상장 준비, 구글 가격 인하 외",
  "show_notes": "팟캐스트 설명란용 2~3줄 요약",
  "items": [
    {"headline": "한 줄 헤드라인", "url": "https://검증된-1차-출처"}
  ],
  "dialogue": [
    {"speaker": "host", "text": "안녕하세요, AI 데일리입니다…"},
    {"speaker": "expert", "text": "네, 오늘 첫 소식은…"}
  ],
  "discord_text": "**📰 오늘의 AI 브리핑 (6/19)**\n\n**1. 헤드라인**\n- 핵심 3줄…\n🔗 출처"
}
```
- `discord_text`는 Discord 마크다운. 각 건 헤드라인+핵심+검증된 출처 링크. 1900자 이내.
- 작성 후 반드시 `python3 publish.py content.json` 실행.
